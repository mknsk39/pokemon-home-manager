# 機能要件 (MVP)

> [!NOTE]
> 基本情報は [README.md](../../README.md) を参照。

## 1. 所持管理

### 1.1 シンプル所持トグル（実装済み）

- 各ポケモンの姿（PokemonForm）ごとに「所持 / 未所持」の2値を管理
- すがたモードでカードをタップするとトグル
- 種別モードでは所持数/全フォーム数（例: `2/5`）を集計表示
- 所持フィルター（所持のみ / 未所持のみ）で絞り込み可能
- データは Firestore `userOwnership/{userId}` に `ownedFormIds: number[]` として保存

### 1.2 ハイブリッド・チェックシステム（将来拡張）

- **個別管理**: 「各ソフトでの所持（例：SV で捕獲済み）」と「HOME 転送済み」を個別に管理
- **除外ロジック**: 「HOME 転送済み」にチェックを入れると、自動的に全体の「未所持リスト」から除外

## 2. オフライン対応

- Firestore の永続化機能を有効化し、オフライン時でも閲覧およびチェック状態の更新が可能
- 次回オンライン時に自動同期

## 3. 認証機能

- Google ログインによる複数端末間データ同期
- 認証後はヘッダーからログアウト（サインアウト）可能
